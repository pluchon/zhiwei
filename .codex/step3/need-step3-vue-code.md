# 校园设施报修与资产管理系统第三阶段 Vue3 前端设计

> 本文档只明确第三阶段前端核心页面与交互。

## 1. 前端规范

- Vue 文件只保留 `<template>` 与外部资源声明。
- 脚本使用同名 `.js`，有私有样式时使用同名 `.scss`。
- 页面通过 `src/api` 调用后端，禁止直接调用 Axios。
- 状态文本和标签使用共享枚举或组件。
- 前端控制可见性与交互，后端负责最终鉴权和并发校验。

## 2. 资产管理页面

新增管理员页面：

```text
views/admin/asset-category
views/admin/asset
```

资产分类页面：

- 支持新增、修改、启用和停用。
- 停用前二次确认；存在未删除资产时展示后端拒绝原因。
- 不展示分类删除和恢复入口。

资产台账页面：

- 支持资产编号、名称、分类、校区和楼栋筛选。
- 支持新增、修改、停用、逻辑删除和恢复。
- 资产图片最多上传一张。
- 资产存在未结束工单时，禁用修改、停用和删除入口。
- 关键操作二次确认，失败时展示后端原因。

## 3. 资产查看与关联报修

新增或复用资产列表与详情页面：

- 管理员和维修师傅可查看全部未删除资产。
- 学生和教师只查看使用中资产。
- 资产详情展示基础信息和允许查看的维修历史。

改造报修创建页面：

- 支持普通报修和资产报修切换。
- 资产报修必须选择资产，并自动带入资产位置。
- 草稿阶段允许更换资产；提交后不显示修改资产入口。
- 资产已有未结束工单时展示明确提示并禁止提交。

改造工单详情：

- 展示资产编号、名称、分类和位置快照。
- 管理员和维修师傅可从资产入口查看完整维修历史。
- 学生和教师仅查看本人关联工单。

## 4. 管理统计与工单导出

改造管理员看板：

- 展示故障类型、校区、楼栋分布和当前状态数量。
- 支持最近 7、30、90 天，默认 30 天。
- 楼栋只展示前十名与“其他”。

改造管理员工单列表：

- 增加资产编号、资产名称和已导出状态筛选。
- 增加固定格式导出按钮。
- 导出超过 1000 条时提示缩小筛选范围。
- 导出期间禁用按钮；成功后刷新已导出标记。
- 已导出工单仍允许再次导出。

## 5. 站内信与维修师傅建议

将现有通知入口调整为统一站内信入口：

```text
系统通知
维修师傅建议
```

维修师傅建议页面：

- 支持提交、查看、撤回、编辑并重新提交本人建议。
- 仅待处理建议可撤回；撤回后进入编辑状态。
- 已采纳和未采纳建议只读。

管理员建议页面：

- 支持按状态和分类筛选。
- 处理时选择采纳或未采纳，并填写必填回复。

## 6. SSE 实时通知

- 登录成功后建立 SSE 连接，退出登录后关闭。
- 收到通知变化事件后刷新未读提示和当前站内信列表。
- 使用浏览器原生重连，不使用 WebSocket 或定时轮询。
- 使用兼容现有认证方式的 SSE 客户端，禁止将 JWT 放入 URL 查询参数。
- SSE 断线不阻止页面正常查询和业务操作。
- 会话失效时停止重连并进入现有登录处理流程。
