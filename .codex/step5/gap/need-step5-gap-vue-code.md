# 第五阶段 AI 模块最终收尾 Vue3 设计

> 本文档只明确前端核心页面与交互。实现必须遵守 `.codex/codex.md` 与 `need-step5-gap.md`。

## 1. 前端规范

- Vue 文件仅保留模板与外部资源声明，脚本和样式使用同名文件。
- 页面统一通过 `src/api` 调用后端。
- 前端只控制交互可见性，权限和数据范围以后端为准。
- AI 加载、空结果、失败和降级状态必须清晰展示。

## 2. 工单列表与派单弹窗

- 管理员工单列表增加固定“AI 辅助”操作列。
- 仅待匹配、待接单工单显示“AI 分析”按钮。
- 点击按钮阻止整行跳转，并在独立弹窗展示分析结果和工单编号。
- 手动派单弹窗顶部增加 AI 派单辅助区域，仅主动点击后展示结果。
- 明确提示分析不推荐或排序候选维修师傅。
- AI 分析不自动填写派单说明或选择维修师傅。

## 3. 疑似重复详情

- 疑似重复卡片先展示完整的整体 AI 判定理由，再展示关联推荐表格。
- 判定理由为空时展示人工判断提示。
- 关联表格列名使用“关联推荐理由”。
- 空推荐列表正常展示，不影响整体理由。
- 仅管理员可见。

## 4. 自然语言导出预览

- 导出预览展示导出类型、有效筛选条件摘要和真实预计数量。
- 数量为 0 或超过 1000 时禁用确认导出并展示原因。
- 条件歧义、不支持或越权时展示明确提示，不提供确认按钮。
- 导出确认继续只对管理员显示。

## 5. AI 助手语义搜索

统一 AI 助手新增结构化结果卡片：

```text
工单搜索结果卡片
资产搜索结果卡片
建议搜索结果卡片
```

- 卡片展示必要摘要并提供详情入口。
- 不展示具体相似度分数。
- 管理员可搜索历史工单、资产和全部建议。
- 维修师傅可搜索资产和本人建议。
- 学生和教师无统一 AI 助手入口。
- 欢迎提示补充语义搜索示例。

## 6. 建议相似提醒

- 维修师傅提交或重新提交建议时展示相似内容提醒。
- 对本人相似建议可展示标题、状态和详情入口。
- 对他人相似建议只展示存在相似内容的提示。
- 提醒不得阻止继续提交。

## 7. API 组织

- 保留现有 `api/asset/import.js`，不新增资产导入 AI API 文件。
- 扩展现有 AI 助手、工单 AI 和建议 API 模块。
- 页面不得直接调用 Axios。

## 8. 前端验证

- 管理员列表、详情和派单弹窗 AI 交互正确。
- 导出预览禁用条件和实际导出一致。
- 三类语义搜索卡片、详情跳转和角色可见性正确。
- 建议相似提醒不泄露他人内容且不阻止提交。
- 前端生产构建通过。
