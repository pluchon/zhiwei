# 第五阶段 AI 模块最终收尾数据库设计

> 本文档只明确最终收尾所需数据库变更。Flyway 只新增迁移，不修改历史迁移。

## 1. 工单关联推荐生成标记

`repair_order` 新增：

```text
ai_link_generated TINYINT NOT NULL DEFAULT 0
```

- `0` 表示从未生成关联推荐，`1` 表示已经生成过。
- 首次生成成功、生成空推荐或降级生成后均置为 `1`。
- 管理员解除全部关联后不得重置。
- 历史工单默认 `0`。

## 2. 索引

- 复核重复报修候选查询所需索引，覆盖故障类型、状态、创建时间、位置和关联资产条件。
- 不为 Embedding 新增向量字段或向量索引。
- 不新增专用向量表。

## 3. Redis 数据

以下数据仅使用 Redis，不落数据库：

- Embedding 向量缓存；
- 自然语言导出预览完整查询条件；
- AI 助手短期会话上下文。

缓存必须设置有效期，不保存密钥、完整模型响应或其他敏感明文。

## 4. 数据兼容

- 新迁移版本按当前最大 Flyway 版本递增。
- 已有 AI 关联、审计、导入和助手数据结构继续复用。
- 不修改或清理历史工单、关联记录和审计记录。
