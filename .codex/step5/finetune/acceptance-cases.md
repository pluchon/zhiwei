# 最小微调验收用例

## 资产识别

| 编号 | 输入 | 期望 |
|---|---|---|
| AR-01 | 标准 Excel 行（名称+分类+位置） | 专用模型返回结构化 JSON，分类可映射 |
| AR-02 | 图片识别（设备铭牌/现场照片） | 视觉模型返回可解析字段 |
| AR-03 | 专用模型不可用 | 降级到基座视觉/对话模型，导入流程不中断 |

## 重复报修

| 编号 | 输入 | 期望 |
|---|---|---|
| DR-01 | 同位置同类故障候选 | 判定 suspected=true，并给出管理员可读理由 |
| DR-02 | 同位置不同故障 | 判定 suspected=false |
| DR-03 | 专用模型不可用 | 降级到基座模型，提交不阻断 |

## 关联推荐（ORDER_LINK）

| 编号 | 输入 | 期望 |
|---|---|---|
| OL-01 | Top3 候选存在且 AI 可用 | 返回逐条推荐理由，仅插入 recommended=true 的关联 |
| OL-02 | AI 判定全部不适合 | 不插入关联，但 ai_link_generated=1 |
| OL-03 | AI 失败 | 展示 Top3 与统一降级提示 |

## 记录要求

- 对比基座模型与专用模型的通过率
- 记录降级触发场景与业务是否中断
